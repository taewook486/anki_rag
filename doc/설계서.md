## 1. 프로젝트 개요

### 1.1 문제 정의

Anki는 플래시카드 기반 간격 반복(Spaced Repetition) 학습 도구로, 사용자가 수년간 축적한 24,472개의 영어 학습 데이터(단어, 구동사, 예문)가 `.apkg` 패키지 파일 형태로 존재한다.

그러나 기존 Anki는 다음 한계를 가진다.

- 자연어 질의 불가능: "포기하다라는 뜻의 구동사를 모두 보여줘" 같은 질문에 답하지 못한다.
- 시맨틱 검색 불가능: 의미적으로 유사한 단어 탐색이 불가능하다.
- 맥락 기반 학습 불가능: AI가 컨텍스트를 종합하여 학습 질문에 답변해 주지 않는다.

### 1.2 해결 방안

RAG(Retrieval-Augmented Generation) 아키텍처를 적용하여 Anki 데이터를 벡터 DB에 인덱싱하고, 하이브리드 검색 + LLM 응답 생성을 통해 지능형 영어 학습 도우미를 구현한다.

### 1.3 핵심 사용 시나리오

| 시나리오 | 입력 예시 | 기대 결과 |
|---------|---------|---------|
| 영단어 검색 | "abandon의 뜻은?" | 정의 + 예문 + 발음 정보 반환 |
| 예문 검색 | "포기하다 영어 예문" | 관련 예문 + 출처 반환 |
| 유사어 검색 | "give up 관련 구동사" | 의미적으로 유사한 구동사 목록 |
| 복합 질의 | "TOEFL 경제 관련 학술 단어" | 필터 + 시맨틱 검색 병합 결과 |

### 2.1 Anki 파일 포맷 구조

```
.apkg (ZIP 아카이브)
├── collection.anki21   (신규 버전 SQLite, 우선 사용)
├── collection.anki2    (구버전 SQLite, fallback)
├── media               (JSON: {"0": "aban.mp3", "1": "give_up.mp3", ...})
├── 0                   (바이너리 오디오 파일)
├── 1
└── ...

SQLite notes 테이블
├── id        (note ID)
├── flds      (필드 값, chr(31) = \x1f 로 구분)
└── mid       (model/note-type ID)

SQLite col 테이블
├── models    (JSON: 노트 타입별 필드명 정의)
└── decks     (JSON: 덱 이름 정의)
```

### 2.2 노트 타입별 필드 매핑

| Note Type | word | meaning | pronunciation | example |
|-----------|------|---------|---------------|---------|
| Simple Model (toefl/xfer) | Question | Answer (HTML) | regex 추출 | 없음 |
| Basic/forvo 계열 | Front | 뜻 | 발음 | 예문 / 예문 뜻 |
| sentences (txt) | sentence | translation | 없음 | 없음 |

## 4. 핵심 모듈 설계

### 4.1 데이터 모델 (`src/models.py`)

```python
class Document(BaseModel):
    word: str
    meaning: str
    pronunciation: Optional[str] = None
    example: Optional[str] = None
    example_translation: Optional[str] = None
    source: str                          # toefl, xfer, hacker_toeic, ...
    deck: str
    tags: list[str] = []
    note_type: Optional[str] = None
    audio_path: Optional[str] = None     # data/media/{source}/파일.mp3

class SearchResult(BaseModel):
    document: Document
    score: float                         # RRF fusion score
    rank: int
```

### 4.2 파서 (`src/parser.py`)

**AnkiParser** — .apkg → List[Document]

- ZIP 추출 후 `collection.anki21` 우선 탐색, 없으면 `collection.anki2` fallback
- `notes.flds`를 `\x1f` (chr 31)로 분리하여 필드 추출
- `col.models` JSON으로 노트 타입별 필드명 매핑
- `[sound:파일명]` 정규식으로 오디오 파일 식별 후 `data/media/{source}/`에 추출
- HTML 태그 BeautifulSoup으로 제거

**TextParser** — tab-separated txt → List[Document]

- `utf-8-sig` 인코딩으로 UTF-8 BOM 자동 처리
- 탭 구분, 최소 2개 필드 보장
- 빈 줄 및 불완전 행 자동 스킵

### 4.3 임베더 (`src/embedder.py`)

**BGEEmbedder** — Document → EmbeddingResult

| 항목 | 값 |
|------|-----|
| 모델 | `BAAI/bge-m3` (FlagEmbedding 라이브러리) |
| Dense 벡터 | 1024차원 float32 (cosine similarity) |
| Sparse 벡터 | SPLADE 방식 token weights `Dict[int, float]` |
| 배치 크기 | 기본 32개 |
| 디바이스 | CUDA 자동 감지, 없으면 CPU fallback |
| 임베딩 텍스트 | `{word} {meaning} {example}` 조합 |

```python
output = model.encode(
    texts,
    return_dense=True,
    return_sparse=True,
    return_colbert_vecs=False,
)
# dense_vecs[i]: shape (1024,)
# lexical_weights[i]: Dict[token_id, weight]
```

### 4.4 인덱서 (`src/indexer.py`)

**QdrantIndexer** — EmbeddingResult → Qdrant

- Named Vectors 스키마: `dense` (cosine) + `sparse` (SparseIndexParams)
- 페이로드: word, meaning, pronunciation, example, source, deck, tags, audio_path
- `upsert()` — 중복 시 덮어쓰기 (idempotent)

```python
vectors_config = {
    "dense": VectorParams(size=1024, distance=Distance.COSINE),
}
sparse_vectors_config = {
    "sparse": SparseVectorParams(index=SparseIndexParams(on_disk=False)),
}
```

> **QdrantClient 초기화 주의사항**
> `location=` 파라미터는 `:memory:` 또는 HTTP URL 전용이다. 로컬 디렉토리 경로는 반드시 `path=` 파라미터를 사용해야 한다.

```python
# 올바른 초기화
if location == ":memory:" or location.startswith("http"):
    self.client = QdrantClient(location=location)
else:
    self.client = QdrantClient(path=location)  # 로컬 파일 경로
```

### 4.5 리트리버 (`src/retriever.py`)

**HybridRetriever** — query str → List[SearchResult]

RRF (Reciprocal Rank Fusion) 알고리즘:

```
score(doc) = Σ  1 / (rank_i + k)
             i ∈ {dense, sparse}

k = 60  (기본값, 상위 랭크에 과도한 가중치 방지)
```

- Dense 검색: `query_points(using="dense", query=query_dense_vector)`
- Sparse 검색: `query_points(using="sparse", query=SparseVector(...))`
- 두 결과를 RRF로 병합 후 상위 K개 반환
- `source_filter`: Qdrant `FieldCondition` 필터 적용

### 4.6 RAG 파이프라인 (`src/rag.py`)

**RAGPipeline** — question str → answer str

```
1. HybridRetriever.search(question, top_k=5)
2. 검색 결과를 구조화된 컨텍스트로 조합
3. 시스템 프롬프트 + 컨텍스트 + 질문을 LLM에 전달
4. 답변 반환 (출처 정보 포함)
```

**LLMProvider 추상화** (교체 가능한 구조):

```
ANTHROPIC_API_KEY 존재? ─Yes─► AnthropicProvider
        │ No
        ▼
LLM_API_KEY 존재? ────────Yes─► OpenAICompatibleProvider
        │ No                     (GLM, OpenAI, OpenRouter 등)
        ▼
ValueError: "LLM_API_KEY 또는 ANTHROPIC_API_KEY 필요"
```

### 4.7 오디오 플레이어 (`src/audio.py`)

플랫폼별 분기:

| OS | 구현 |
|----|------|
| Windows | `playsound` 라이브러리 |
| macOS | `afplay` subprocess |
| Linux | `aplay` / `paplay` subprocess |

- `audio_path=None` 또는 파일 미존재 시: 예외 없이 경고 메시지 출력 후 계속 진행

## 5. 기능 요구사항

### REQ-001: Anki 패키지 파싱

**WHEN** 사용자가 `.apkg` 파일 경로를 제공하면
**THE SYSTEM SHALL** 해당 파일을 ZIP으로 열고 내부의 SQLite DB를 파싱하여 노트 목록을 반환한다.

- `.anki21` 우선, 없으면 `.anki2` fallback
- `flds`를 `\x1f`(chr 31)로 분리
- `col.models` JSON에서 노트 타입별 필드명 매핑

### REQ-002: 텍스트 파일 파싱

**WHEN** 사용자가 탭 구분 텍스트 파일을 제공하면
**THE SYSTEM SHALL** UTF-8 BOM으로 읽고 탭 분리하여 문장-번역 쌍을 반환한다.

### REQ-003: BGE-M3 임베딩

**WHEN** Document 배치가 준비되면
**THE SYSTEM SHALL** `BAAI/bge-m3` 모델로 Dense(1024차원) + Sparse 벡터를 동시에 생성한다.

### REQ-004: Qdrant 인덱싱

**WHEN** 임베딩이 완료되면
**THE SYSTEM SHALL** Qdrant에 Dense+Sparse 벡터와 메타데이터를 저장한다.

- 컬렉션명: `anki_rag` (설정 가능)
- 페이로드: word, meaning, pronunciation, example, source, deck, tags, audio_path

### REQ-005: 하이브리드 검색 (RRF Fusion)

**WHEN** 사용자가 검색 쿼리를 입력하면
**THE SYSTEM SHALL** Dense + Sparse 검색 결과를 RRF로 병합하여 상위 K개를 반환한다.

- RRF 수식: `score = Σ 1/(rank + 60)`
- 필터 지원: `source`, `deck` 필드

### REQ-006: Claude/OpenAI 호환 RAG 응답 생성

**WHEN** 검색 결과가 준비되면
**THE SYSTEM SHALL** LLM API를 통해 컨텍스트 기반 자연어 답변을 생성한다.

- 우선순위: ANTHROPIC_API_KEY > LLM_API_KEY (OpenAI 호환)
- 출처(source, deck) 정보 포함

### REQ-007: 오디오 파일 추출

**WHEN** `.apkg` 파일을 파싱할 때
**THE SYSTEM SHALL** ZIP 내 미디어 파일을 `data/media/{source}/` 디렉토리에 추출한다.

- `media` JSON: `{"0": "aban.mp3", ...}`
- `[sound:파일명]` 패턴 정규식 추출
- 이미 추출된 파일은 재추출 스킵 (idempotent)

### REQ-008: CLI 인터페이스

```bash
python -m anki_rag index   --data-dir ./data
python -m anki_rag search  "abandon" --source toefl --top-k 5
python -m anki_rag search  "abandon" --play-audio
python -m anki_rag query   "abandon의 뜻과 예문을 알려줘"
python -m anki_rag chat
```

## 8. LLM 프로바이더 설계

### 8.1 설계 배경

초기 설계는 Anthropic API에 고정되어 있었으나, OpenAI 호환 API(GLM 등)를 지원하도록 프로바이더 추상화를 도입하였다.

### 8.2 LLMProvider 프로토콜

```python
class LLMProvider(Protocol):
    def generate(self, prompt: str, stream: bool = False) -> str: ...
    def stream(self, prompt: str) -> Iterator[str]: ...
```

구현체:
- `AnthropicProvider` — Anthropic SDK 래퍼
- `OpenAICompatibleProvider` — OpenAI SDK 래퍼 (GLM, OpenRouter, OpenAI 등)

### 8.3 환경변수 설정

```bash
# OpenAI 호환 API
LLM_BASE_URL=https://your-endpoint/v1
LLM_API_KEY=your-api-key
LLM_MODEL=gpt-4o-mini

# Anthropic API (선택, 우선순위 높음)
ANTHROPIC_API_KEY=sk-ant-...
```

### 8.4 의존성 변경

| 항목 | 이전 | 이후 |
|------|------|------|
| 핵심 의존성 | `anthropic>=0.28` | `openai>=1.0` |
| 선택 의존성 | 없음 | `anthropic>=0.28` (optional) |
